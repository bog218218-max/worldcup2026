# Product

## Register

product

## Users

Friends participating in a private World Cup prediction tournament. They use Telegram for all actions that change data, and the website as a public scoreboard they can open during matches from desktop or mobile.

## Product Purpose

The service tracks paid participants, match predictions, scoring, prizes, leaderboard movement, match details, player profiles, and tournament statistics. Success means a small group can run the tournament without spreadsheets: participants submit predictions in Telegram, an admin enters results, and the website updates read-only public data.

## Brand Personality

Sharp, competitive, trustworthy. The interface should feel like a compact sports operations room, not a marketing site.

## Anti-references

Avoid generic SaaS landing pages, oversized hero sections, decorative gradients, playful betting-casino styling, glassmorphism, and any UI that makes the tournament feel less credible or harder to scan.

## Design Principles

- Make the current standings and prize race obvious within seconds.
- Keep data dense, but preserve clear hierarchy for mobile reading.
- Use color for state and meaning: leader, prize zone, exact scores, misses, live matches.
- Treat Telegram as the control plane and the website as a read-only public scoreboard.
- Prefer predictable sports dashboard patterns over novelty.

## Accessibility & Inclusion

Target WCAG AA contrast. Do not rely on color alone for result types or match status. Respect reduced motion preferences and keep controls and tables usable on mobile screens.
